User Documentation

Installation:

Step 1: compile the GetOnePic.java and PictureBrowse.java in
your_home_dir/catalina/webapps/proj1/WEB-INF/classes
Step 2: start sqlplus load files: setup_new.sql, data.sql, triger.sql,
myIndex.sql, updateIndex.sql
Step 3: run 'starttomcat' in terminal
Step 4: Use the following address in your browser to access the html file:
http://ui???.cs.ualberta.ca:"your-first-port-number"/proj1/oracle_login.html
Step 5: input your oracle database account information into the webpage

Login:

-----either your user name or password is not supposed to longer
than 24 characters,then click the Login button.
-----one can modify his/her account password and personal information with saving changes

User Management:

Create a new user:

Personal Information:make sure the user's personal information is alread existed if you want to skip this section.
Otherwise, you need to enter the user's personal information according the format:
-----either your first name or last name suppose no longer than than 24 characters
-----your personal id must be integer
-----either your address or email is not supposed to longer than 128 characters
-----your phone number is not supposed to longer than 10 integers
Account Information:if you skipped the personal information page, then you can enter existed person ID.
After that, please enter the new user account information.
Family Doctor:You can build a doctor-patient relationship or skip this page.
Click "finish adding" to confirm the creation.

Update user information:
Search:a valid user name is required before any updating
Updating:-----either your first name or last name suppose no longer than than 24 characters
-----your personal id must be integer
-----either your address or email is not supposed to longer than 128 characters
-----your phone number is not supposed to longer than 10 integers
-----one can rebuild the current user's doctor-patient relationship

Report Generation:

-----A Diagnosis shorter than 128 characters is required
-----A time interval is required!(from a earlier date to a later date).Submit afterward

Uploading:

A valid relationship between the given patient and doctor is required while the rest is not.
The uploading part allows user to upload one or more images for the current record

Search Engine:

-----a user can search by a key word or a list of key words(seperated by ",")
-----one can view a list of medical images(thumbnails) of a record by clicking the link
"Pictures", which can be further viewed by clicking the images themselves(normal size)
or the links below them(full size).

Data Analysis:

Analysis by particular value(s):-----All values are optional, you can leave some values blank or default if you don't want
to specify those values
-----A time interval should from a earlier date to a later date
Analysis by attribute(s):-----All attributes are optional, you can leave some attribute blank or default if you don't want
to specify those attribute
